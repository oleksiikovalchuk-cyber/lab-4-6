ОПИС ВИКОНАНИХ КРОКІВ

У ході виконання лабораторної роботи було реалізовано наступні етапи:

Створено новий компонент сторінки ContactView.vue у папці src/views.

Налаштовано маршрутизацію: у файл router/index.ts додано новий об'єкт маршруту з шляхом "/contact" та прив'язкою до створеного компонента.

Оновлено головне меню навігації у файлі App.vue шляхом додавання компонента RouterLink для переходу на сторінку контактів.

Створено компонент картки користувача UserCard.vue у папці src/components.

У компоненті UserCard застосовано TypeScript (defineProps) для суворої типізації вхідних даних: username (рядок) та isActive (булеве значення).

Компонент UserCard успішно імпортовано та використано на сторінці ContactView з передачею різних параметрів.

ТЕКСТ ПОМИЛКИ (ЧАСТИНА 4)

При спробі передати числове значення (наприклад, 123) у пропс isActive, який очікує булевий тип, середовище розробки та TypeScript видають наступну помилку невідповідності типів:

Type 'number' is not assignable to type 'boolean'.

Також у консолі браузера з'являється попередження від Vue:
[Vue warn]: Invalid prop: type check failed for prop "isActive". Expected Boolean, got Number with value 123.

ВИСНОВКИ

Використання TypeScript у проєкті забезпечує:

Статичну типізацію, що дозволяє виявляти помилки ще на етапі написання коду (наприклад, передачу некоректних типів даних у пропси).

Покращене автодоповнення коду (IntelliSense) та документацію "на льоту", що прискорює розробку.

Більш надійний та зрозумілий код, який легше підтримувати в майбутньому.

Використання Vue Router дозволяє:

Створювати односторінкові додатки (SPA), де переходи між сторінками відбуваються миттєво без перезавантаження сторінки.

Організувати зручну навігацію та структуру посилань у додатку.

Зберігати історію переходів браузера, забезпечуючи коректну роботу кнопок "Назад" і "Вперед".
